# Wkład w projekt (Contributing)

Dzięki, że chcesz pomóc w rozwoju projektu!

## Ważne: osobny backend
Backend jest osobnym projektem: **[Tsugumik/shinden-pl-api-rs](https://github.com/Tsugumik/shinden-pl-api-rs)**.  
Issues i PR-y dotyczące backendu zgłaszaj w tamtym repozytorium.

## Jak zacząć
1. Zrób fork repozytorium.
2. Utwórz nową gałąź z opisową nazwą (np. `feature/playlist`, `fix/episode-status`).
3. Wprowadź zmiany i upewnij się, że wszystko działa.
4. Zrób PR do gałęzi domyślnej projektu.

## Pull Requesty
- Opisz **co** zmieniasz i **dlaczego**.
- Jeśli PR naprawia problem, dodaj numer issue (np. `Closes #12`).
- Utrzymuj PR-y możliwie małe i skupione na jednym celu.
- Upewnij się, że kod jest czytelny i spójny ze stylem projektu.

## Styl i jakość kodu
- Stosuj się do istniejącego stylu w repozytorium.
- Unikaj dużych refaktorów bez wcześniejszego uzgodnienia.
- Jeśli dodajesz nowe funkcje przetestuj wcześniej ich działanie.

## Komunikacja
- Jeśli nie jesteś pewien kierunku zmian - otwórz dyskusję albo skonsultuj pomysł w PR.

Dzięki za wkład!
